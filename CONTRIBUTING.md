# Contributing to FastInBox

Obrigado por contribuir com o ecossistema FastInBox.

## Repositorios

- `front`: experiencia web e jornadas de usuario
- `back`: API e regras de negocio
- `docs`: documentacao tecnica no GitHub Pages
- `.github`: padroes de org, workflows e templates

## Fluxo de trabalho recomendado

1. Abra uma issue usando os templates
2. Crie branch curta e descritiva
3. Mantenha commits pequenos e semanticos
4. Abra PR com checklist completo
5. Vincule issue no PR (ex.: `Closes #123`)

## Padrao de qualidade

- Evite alterar escopo sem registrar na issue
- Priorize menor diff possivel sem regressao
- Rode validacoes locais antes de abrir PR
- Atualize docs quando houver impacto funcional

## Convencao de branch

- `feat/<descricao-curta>`
- `fix/<descricao-curta>`
- `docs/<descricao-curta>`
- `chore/<descricao-curta>`

## Convencao de commit

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `chore: ...`
- `refactor: ...`
